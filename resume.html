Wikispeedia

The prettiest Wikipedia game on the internet.

The Wikipedia game is an old standby for bored teens, but it's rarely done right. For our final project for CS 32 at Brown, Mckenna Cisler, Rohan Jha, Sean Hastings and I decided to perfect this age-old game. I worked on the user experience, including lobby experience, site-wide design, and graphic design. I also set up hosting so anyone can play online.

EQUiSat

Making space accessible.

At Brown Space Engineers, we are dedicated to making space more accessible for people of all backgrounds. Following our schematics, anyone can make a satellite for only $3,776. Successfully deployed on July 13th, 2018, EQUiSat is the first satellite to test LiFePO batteries in space and its four LEDs are visible by the naked eye. I have worked on organizing satellite operations using FreeRTOS and optimizing data transmission, especially error packaging.

I am now project manager and co-president, so in addition to coordinating satellite operations and managing our over 75 members, I am also leading the design of our next CubeSat. I am responsible for effective communication with NASA, the University, and any other external entity.

For more information, you can visit the team website here. View our team GitHub here.

  • Date: September 2016 - Present
  • Categories: Systems, Avionics

The Social Playlist

Make the party's playlist communal.

Ever wanted to play music at a party but allow your friends to pick some of the music? The Social Playlist attempts to solve that problem by allowing a host to set up a room which visitors can join. Visitors can suggest songs (which are automatically queued on YouTube) which the host can rearrange or reject. We also attempted to use Microsoft's Emotion API to suggest a playlist based on the mood detected by a device's webcam. This project was created at Hack@Brown 2017. I made all user-facing components and attempted to integrate Microsoft's Emotion API.

For installation directions and more information, view our project git here.

  • Date: February 2017
  • Categories: UI/UX, Frontend, Web Development

Rocket Filtering

A searching/filtering interface using React JS.

This website, written for CSCI 1300: UI/UX, demonstrates a number of the principles learned in that class. First, it's clear that the only controllable sections of the interface are at the top of the page. Filters are on the top left, then sorting, then search is on the top right. These are fairly standardized positions for the respective components. The grid layout make it easier for users to see more information than a linear layout would have been. These features work to promote the central goal of the site - comparing rockets by a variety of criteria. With more information this site could be useful for space mission managers to compare and pick rockets for launches.

The project is avaliable on GitHub.

  • Date: November 2018
  • Categories: React, Javascript, UI/UX

I was a Software Engineering Intern on Intune's mobile app management (MAM) team for Android. They develop an SDK that allows IT pros to make tightly managed apps. I made an example app demonstrating how to use this SDK called Taskr. Taskr is open source and available on GitHub.

National Weather Service

Redesigning an old but useful website.

Despite having the best weather information available online, the National Weather Service's website looks like it hasn't been updated since the 1990s. I check it most days, and almost every time I do I'm frustrated by the terrible interface. So when we chose websites to redesign for CSCI 1300: UI/UX, I knew I wanted to choose it. By simplifying the interface, a move that also makes it look more modern, I believe I made the information significantly more accessible. This is the interface that the National Weather Service should have for the 21st century.

The project is available on GitHub.
